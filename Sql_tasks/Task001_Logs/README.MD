**Контекст:** 

Есть высоконагруженная таблица логирования действий пользователей в БД (например, PostgreSQL или MS SQL Server):
`UserLogs (Id INT, UserId INT, Action VARCHAR(100), CreatedAt DATETIME)`
Таблица огромная — около 100 млн записей.

**Задача:** 
1.  Напиши SQL-запрос, который возвращает `UserId` тех пользователей, которые совершили **более 5 действий** типа `'Click'` за **последние 30 минут**.
2.  Напиши `CREATE INDEX` запрос для создания оптимального индекса, который позволит СУБД выполнить этот запрос через `Index Seek` (быстрый поиск), избегая `Table Scan` (полного сканирования таблицы). Объясни выбор полей в индексе.
